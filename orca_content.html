ORCA - An OpenSource Orchestration Solution for SONiC

ORCA is an open-source orchestration solution for SONiC. It provides a user-friendly interface for managing and configuring SONiC devices. ORCA is built on top of gNMI and uses a Neo4j graph database to store network topology. ORCA is designed to be scalable and extensible, and it can be used to manage large and complex SONiC networks. It is easy to install and use, and it provides a comprehensive set of features for managing SONiC devices. ORCA is a valuable resource for the SONiC community. It is an Open Source project that is constantly being improved and expanded.

  • ORCA Quick Start

    ORCA (backend and UI) can be installed within couple of minutes. All is needed is to start 3 docker containers. Refer quick start sections at following links :
    • Run Neo4j and orca_backend docker containers.
    • Run orca_ui docker container.

  • ORCA Architecture

    • ORCA has following components-
      • ORCA Network Library - South bound gNMI interface to SONiC Devices.
      • ORCA Backend - Django based REST wrapper for ORCA Network Library.
      • ORCA UI - GUI created usign REST APIs provided by ORCA Backend.
      • ORCASK - AI Enabled chatbot for SONiC configuration

    • ORCA High Level Design

    • ORCASK - AI Enabled chatbot for SONiC configuration

      ORCA has AI enabled chatbot called ORCASK, ORCASK converts user queries to ORCA API calls. ORCASK currently can be configured to use 2 LLMs - GPT and Llama. GPT models can only be used in cloud, Llama on the other hand can be locally hosted.

  • ORCA Features

    • Network Discovery -

      ORCA Discovers the network topology using LLDP Neighbor Links. User can provide management IP address of one of the device in the network. Using a single IP address, ORCA identifies LLDP neighbors of the seed device then neighbors of neighbor and so on until ORCA discovers the network topology of the all connected devices. Discovery can also be triggered by providing the network address. In the graphical representation device is the parent node and all its components and network configuration is the child node, connected with a specific relationship.

      What ORCA discovers-

      • Device Details
      • Device Interfaces.
      • LLDP Neighbor Links
      • Port Groups
      • MCLAG with peer links
      • BGP and Neighbor links
      • VLAN
      • Port Channels

    • Neo4j Topology Database -

      ORCA stores the network topology in Neo4j graph database (community edition). Where the neo4j nodes in the database represents device components and network configurations and neo4j relationships represents the links between them. ORCA updates the topology in the database using gNMI subscriptions and for some configuration doing rediscovery of specific device components.

    • Device Configuration -

      ORCA uses gNMI to configure the network topology. Using ORCA following configurations can be done.

      • Interfaces
      • Port Groups
      • MCLAG
      • BGP
      • VLAN
      • Port Channels
